# Specifications

This folder contains application specifications, standards, and architectural documentation.

## Purpose

The `.specs/` folder houses:
- Technical specifications and requirements
- API documentation and contracts
- Database schemas and migrations
- Architecture decision records (ADRs)
- Coding standards and conventions
- Integration specifications

## Organization

```
.specs/
├── README.md           # This file
├── architecture/       # System architecture docs
├── api/               # API specifications
├── database/          # Schema docs, migrations
├── standards/         # Coding standards
└── decisions/         # Architecture decision records
```

## When to Update

- New features are planned
- Architectural changes are proposed
- Standards evolve
- API contracts change

## Guidelines

- Keep specifications up to date with implementation
- Use clear, unambiguous language
- Include diagrams where helpful
- Link to related `.plans/` and `.agents/` docs
